### About the Repository

The BLOCKDEF Community Projects repository is a collaborative platform where developers, researchers, and enthusiasts can contribute security-related content. Our goal is to create a comprehensive resource for understanding, addressing, and mitigating security risks in the blockchain ecosystem.

### How to Contribute

Contributions to this repository can take many forms, including but not limited to:

- **Security Audits**: Share detailed audits of smart contracts or blockchain applications.
- **Vulnerability Reports**: Submit reports on discovered vulnerabilities and their mitigations.
- **Research Papers**: Contribute research on security trends, threats, and defenses.
- **Tutorials and Guides**: Provide educational content on best practices for secure development.
- **Tool Recommendations**: Suggest tools and frameworks that enhance security.

#### Getting Started

<Info>
  **Prerequisite** You should have installed Node.js (version 18.10.0 or
  higher).
</Info>

Step 1. Clone the BLOCKEF repository:

<CodeGroup>

```bash npm
git clone https://github.com/blockdef/community-projects.git
```

</CodeGroup>

Or using GitHub CLI:

```bash
gh repo clone blockdef/community-projects
```

Step 2. Installing dependencies once in the project's root directory, run the following command to install the project's dependencies:

```bash npm
npm install
```

Step 3. After installing the dependencies once in the project's root directory, run the following command to start the project:

```bash npm
npm run dev
```

### Custom Ports

BLOCKDEF uses port 3000 by default. You can use the `--port` flag to customize the port the repository runs on.

For additional resources and guidance, visit our [documentation](https://docs.blockdef.com) and participate in our [community forum](https://blockdef.dev/contribute).


### Specific Guidelines

1. **Content Quality**: Ensure that all contributions are accurate, well-researched, and relevant to security topics in the blockchain and Ethereum ecosystem.
2. **Formatting**: Use clear headings, subheadings, and formatting to enhance readability. Follow Markdown or MDX syntax rules for structuring your content.
3. **Code Examples**: Provide code examples where applicable. Ensure that all code is properly commented and tested.

### Mandatory Fields in `.mdx` Files

When submitting a new `.mdx` file, you must include the following metadata fields at the top of your document. These fields help us manage and categorize content effectively:

```mdx
---
title: Chainlink Integration Guide
date: 2024-05-28
author: BLOCKDEF
github: (github of your repo)
slug: chainlink-integration
headings (table of content):
  - What 
  - The-EVM-Security-Testing-Guide
  - Purpose
  - Design
  - Analyze
  - Fuzzing
  - Deploy
---

The content of your project...
```
